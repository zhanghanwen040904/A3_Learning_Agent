# 系统开发说明书

## 1. 项目概述

A3 Learning Agent 是面向《人工智能导论》课程的个性化学习资源生成与学习多智能体系统。系统围绕第十五届中国软件杯 A3 赛题要求，融合大语言模型、RAG 课程知识库、多智能体协同、多模态生成、AI 辅助编程、内容安全审核和学习效果评估，解决高校学生学习资源繁杂、学习路径不清晰、反馈滞后和个性化辅导不足的问题。

系统采用 Web 应用形式实现，学生可通过浏览器完成登录、画像构建、资源生成、路径规划、智能答疑和学习评估。

## 2. 总体架构

系统分为六层：

1. 用户交互层：Vue 3 + Element Plus，提供现代化 AI 产品交互界面；
2. 业务 API 层：Flask 提供 RESTful 接口；
3. 多智能体协同层：负责画像、检索、生成、规划、答疑、评估和复核；
4. RAG 知识库层：基于《人工智能导论》课程文档集进行可信检索；
5. 数据存储层：MySQL 存储用户、画像、资源、路径、评估和学习事件；
6. AI 能力层：调用讯飞星火、SeeDance 和内容审核服务。

| TextAgent | 文本资源生成 | 画像、知识片段 | 讲解文档、阅读材料 |

| CodeAgent | 实操案例生成 | 课程知识点 | Python 实操案例 |

## 4. 核心业务流程

### 4.1 对话式画像流程

1. 学生输入自然语言学习情况；
2. ProfileAgent 抽取学习特征；
3. 系统生成知识基础、学习风格、薄弱点、学习目标、时间偏好、课程进度六维画像；
4. 画像写入数据库；
5. 后续评估结果会动态更新画像。

### 4.2 多智能体资源生成流程

1. 系统读取学生画像；
2. RetrieveAgent 从课程知识库检索相关教材依据；
3. TextAgent 生成讲解文档和拓展阅读；
4. QuizAgent 生成分层练习题；
5. VisualAgent 生成思维导图；
6. CodeAgent 生成实操代码案例；
7. VideoAgent 生成教学视频/动画任务；
8. SafetyAgent 检查生成内容和引用来源；
9. 资源保存并在前端卡片化展示。

### 4.3 学习路径规划流程

1. 读取画像、学习进度和掌握度；
2. 检索相关课程知识；
3. PathAgent 输出阶段目标、学习顺序、推荐资源、练习方式和评估指标；
4. 路径保存到数据库；
5. 学习评估后可再次生成路径，实现动态调整。

### 4.4 智能答疑流程

1. 学生提交问题；
2. 系统进行输入内容审核；
3. TutorAgent 调用 RAG 检索课程依据；
4. 生成文字解释、图解说明和自测题；
5. SafetyAgent 进行安全和防幻觉复核；
6. 前端以伪流式方式展示回答和来源标签。

### 4.5 学习效果评估流程

1. 学生提交练习答案；
2. EvaluatorAgent 进行自动评分；
3. 写入 quiz_result；
4. 更新 mastery_record；
5. 写入 learning_event；
6. 根据掌握度更新 student_profile 中的薄弱点和课程进度；
7. 输出下一步学习建议。

## 5. RAG 防幻觉机制

系统自建《人工智能导论》10 章课程文档集，所有答疑、路径和资源生成均优先检索课程依据。RAG 模块提供：

- txt/md/pdf/docx 文档读取；
- 文本清洗；
- 文本切片；
- Chroma 向量库构建；
- BGE 中文嵌入模型检索；
- 关键词降级检索；
- 结构化 sources 返回；
- 前端来源展示。

当向量模型不可用或网络环境受限时，系统自动降级为关键词检索，避免演示中断。

## 6. 内容安全机制

系统在以下环节接入内容安全：

1. 学生输入审核；
2. 资源生成输出审核；
3. 智能答疑输出审核；
4. SafetyAgent 结合来源进行复核；
5. 前端展示安全状态和引用来源。

## 7. 前端界面设计

前端使用 Vue 3 + Element Plus，实现以下页面：

- 登录注册；
- 多智能体架构；
- 对话式画像；
- 学习资源；
- 学习路径；
- 智能答疑；
- 学习评估；
- 知识库管理。

界面使用卡片式布局、渐变背景、Markdown 渲染、进度条、步骤条和来源标签，符合现代 AI 产品交互规范。

## 10. 创新点

1. 面向高校课程的多智能体协同学习系统；
2. 将 RAG 与安全复核结合，降低大模型幻觉；
3. 画像、资源、路径、答疑、评估形成学习闭环；
4. 支持向量检索和关键词降级，提升演示稳定性；
5. 将多模态资源、代码案例和题库生成整合到统一学习路径；
6. 前端提供多智能体架构展示页，便于评审理解系统设计。
