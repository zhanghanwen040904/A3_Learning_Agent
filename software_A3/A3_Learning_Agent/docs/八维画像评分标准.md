# 八维学生画像评分标准

本文档用于说明系统中“学生画像”页面的八维雷达图是如何生成的，以及每个维度的评分依据是什么。

## 一、总体原则

当前版本不再使用“字段字数越多分越高”的简单方式，而是采用三层证据综合评分：

1. 对话画像层  
   从用户与智能体的自然对话中抽取专业、课程、目标、薄弱点、偏好等信息。

2. 学习行为层  
   结合会话记录、学习事件、资源生成与资源反馈，判断画像是否稳定。

3. 评估结果层  
   结合练习得分、知识点掌握记录、错题与薄弱知识点，校准画像真实性。

## 二、八个维度

系统当前展示的八个维度为：

- 知识基础
- 认知风格
- 易错点偏好
- 学习目标
- 学习历史
- 课程进度
- 时间节奏
- 资源偏好

## 三、评分区间说明

- 0–64：低
- 65–84：中
- 85–100：高

这里的“高/中/低”不是学生能力绝对水平，而是“系统对该维度的识别充分度 + 证据支撑强度”的综合结果。

## 四、各维度评分依据

### 1. 知识基础

主要依据：

- 是否在对话中明确提到基础情况
- 练习平均得分
- `mastery_record` 中的知识点掌握度
- 是否存在多个掌握度偏低的知识点

解释：

- 如果学生明确说出“这部分会/不会”“基础薄弱/还行”，系统先得到一层基础判断
- 再结合练习成绩与知识点掌握记录进行校准
- 如果低掌握度知识点较多，会下调分数

### 2. 认知风格

主要依据：

- 是否主动表达理解方式偏好
- 是否提到图解、视频、代码、案例、刷题、自学等关键词
- 是否已经积累多轮稳定会话
- 是否存在资源偏好信息

解释：

- 这个维度重点不在“强不强”，而在“风格是否清晰”
- 会话越多、偏好越稳定，评分越高

### 3. 易错点偏好

主要依据：

- 是否主动提到薄弱点
- 是否出现“不会、模糊、混淆、卡住、困难”等表达
- `mastery_record` 中低于 70 分的知识点数量
- 最近练习平均分是否偏低

解释：

- 该维度本质上是“系统对学生薄弱点的识别程度”
- 有明确错题、弱点知识点时，这个维度的可信度会明显提高

### 4. 学习目标

主要依据：

- 是否有明确目标表述
- 目标是否可量化，如分数、考试、阶段目标
- 是否明确专业与目标课程

解释：

- “我要学软件工程”和“我想期末拿到 90 分”不是一个清晰度
- 可量化、可执行的目标会得到更高分

### 5. 学习历史

主要依据：

- 是否描述过往学习经历
- 是否提到学过哪些课程、做过哪些练习、之前掌握到什么程度
- 会话数量
- 练习数量

解释：

- 该维度反映“系统是否掌握学生已有学习背景”
- 如果只有当前问题，没有历史信息，则分数不会高

### 6. 课程进度

主要依据：

- 是否提到章节、阶段、已学/未学内容
- 是否已有练习记录
- 是否已有资源使用记录
- 是否有较强或较弱的知识点群

解释：

- 这个维度强调“学到哪里了”
- 对话、练习和资源行为都能作为进度证据

### 7. 时间节奏

主要依据：

- 是否提到早上、晚上、周末、碎片时间、固定时间等信息
- 是否形成了多轮稳定对话

解释：

- 这个维度不是时间管理能力，而是学习节奏画像
- 明确的时间习惯越多，分数越高

### 8. 资源偏好

主要依据：

- 是否明确偏好视频、文档、题目、代码、案例等资源
- 是否已有资源生成/使用记录
- 是否存在资源反馈数据

解释：

- 该维度反映“系统是否知道该给学生什么形式的内容更合适”
- 如果后续加入点击率、完成率，这个维度还可以继续增强

## 五、当前系统使用到的数据表

评分逻辑当前主要参考以下数据：

- `student_profile`：对话抽取得到的画像字段
- `profile_session`：会话数量、会话轨迹
- `quiz_result`：练习得分记录
- `mastery_record`：知识点掌握度
- `learning_event`：学习行为事件
- `resource_feedback`：资源反馈
- `study_resource`：资源生成/使用记录

## 六、答辩时可以怎么讲

你可以这样介绍：

> 我们的八维画像不是简单由大模型“编出来”的，而是采用“对话抽取 + 学习行为 + 评估结果”三层证据融合。  
> 其中，大模型负责从自然对话中抽取学生特征；系统行为数据负责验证这些特征是否稳定；练习结果与知识点掌握记录则进一步校准画像真实性。  
> 因此，画像不仅能描述学生，还能解释“为什么这样判断”。

## 七、后续还可以继续增强的方向

1. 引入章节完成率，强化“课程进度”维度  
2. 引入资源点击/停留/完成率，强化“资源偏好”维度  
3. 引入错题知识点聚类，强化“易错点偏好”维度  
4. 引入时间序列行为，强化“时间节奏”维度  
5. 将每个维度拆成“识别充分度”和“能力状态”两个子分数
