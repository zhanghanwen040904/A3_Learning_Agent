# 中国软件杯 A3 项目模块拆解与实施计划

## 1. 分工原则

本项目按 20 天（3 周）开发组织，采用三人并行开发、集中联调、最后收尾的方式推进。

分工原则：

1. 胡博涵负责后端基础设施，保证接口、知识库和大模型调用可用，同时负责项目协调。
2. 赵嘉诚负责多智能体核心能力，保证赛题”多智能体系统”亮点成立。
3. 徐英博负责前端和演示材料，保证评审能直观看到完整流程。
4. 第 1 周以并行开发为主，第 2 周起进入集中联调。
5. 所有模块必须围绕《计算机组成原理》课程样例展开。

## 7. 接口协作规范

为避免三人联调时互相等待，所有接口统一使用 JSON 返回。

### 7.1 `/chat/send`

用途：发送用户消息，触发画像、检索、多智能体生成，并支持 SSE 流式返回。

请求示例：

```json
{
  "user_id": "demo_user",
  "message": "我 Cache 映射方式总是分不清，希望用图解和例题学习。",
  "chapter": "Cache 基本原理"
}
```

返回事件类型：

```json
{
  "event": "agent_status",
  "agent": "MindMap Agent",
  "status": "running",
  "message": "正在生成 Cache 映射方式思维导图"
}
```

```json
{
  "event": "content",
  "type": "markdown",
  "content": "Cache 映射方式主要分为直接映射、全相联映射和组相联映射..."
}
```

### 7.2 `/profile/get`

用途：获取学生画像。

返回示例：

```json
{
  "user_id": "demo_user",
  "major": "计算机科学与技术",
  "grade": "大二",
  "weak_points": ["Cache 映射方式", "流水线冲突"],
  "learning_preference": ["图解", "例题", "代码示例"],
  "pace": "每天 1 小时"
}
```

### 7.3 `/resource/generate`

用途：按章节和画像生成资源。

请求示例：

```json
{
  "user_id": "demo_user",
  "knowledge_point": "Cache 映射方式",
  "resource_types": ["doc", "mindmap", "quiz", "code", "video_script"]
}
```

返回示例：

```json
{
  "resources": [
    {
      "type": "doc",
      "title": "Cache 映射方式个性化讲解",
      "content": "..."
    },
    {
      "type": "mindmap",
      "title": "Cache 映射方式思维导图",
      "content": "mindmap json or mermaid"
    }
  ],
  "review": {
    "passed": true,
    "notes": ["内容基于知识库片段 ai-cache-002 和 ai-cache-003"]
  }
}
```
