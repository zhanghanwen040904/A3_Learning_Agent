### XP（极限编程）模型的优缺点
知识路径：软件工程 > 第一章 软件工程学概述（下） > XP（极限编程）模型的优缺点
起始页：110
[第110页] o XP模型克服了传统模型不灵活机动的缺陷，是一种面
[第110页] 向用户场景的轻量级模型。它只适合于中小型开发小组。 o 模型的优点
[第110页] (1)由于采用简单策略，不需要长期计划和复杂管理，因而开发周 期短； (2)由于采用迭代增量开发、反馈修正和反复测试的方法，因而软 件质量有保证； (3)由于适应用户需求的变化，因而与用户关系和谐。 o 模型的缺点

#### (1)XP模型作为一种新的模型，在实际应用中还存在着一些问题， 引起了许多争议。 (2)它只适用于小型项目、小型项目组，不大适用于大型项目、大 型项目组。
