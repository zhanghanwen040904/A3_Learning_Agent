# 测试说明书

## 1. 测试目标

验证 A3 Learning Agent 是否满足 A3 赛题对个性化画像、多智能体资源生成、学习路径规划、智能答疑、学习效果评估、内容安全和提交材料完整性的要求。

## 2. 测试环境

| 项目 | 配置 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| 后端语言 | Python 3.10+ |
| 前端框架 | Vue 3 + Vite |
| 数据库 | MySQL 8.x |
| 浏览器 | Chrome / Edge |
| AI 服务 | 讯飞星火、SeeDance、内容审核；支持 MOCK_AI 演示模式 |

## 3. 后端基础测试

### 3.1 语法检查

```bash
cd backend
python -m compileall .
```

预期结果：无语法错误。

python app.py

{"code":200,"msg":"成功","data":{"status":"ok","service":"A3 Learning Agent"}}

```bash
python -c "from db import mysql_db; print(mysql_db.query_one('SELECT 1 AS ok'))"
```

预期结果：

```json
{"ok": 1}
```

## 4. 前端构建测试

```bash
cd frontend
npm install
npm run build
npm run dev
```

预期结果：构建成功，浏览器访问 `http://localhost:5173` 可打开系统。

## 5. 功能测试用例

### 5.1 登录注册

| 步骤 | 输入 | 预期结果 |
| --- | --- | --- |
| 注册账号 | username=test password=123456 | 注册成功 |
| 登录账号 | username=test password=123456 | 返回 token 并进入系统 |
| 未登录访问业务页 | 清除 localStorage token | 自动跳转登录页 |

### 5.2 对话式画像

输入：

```text
我是计算机专业大二学生，学过 Python 和高等数学，但人工智能基础比较薄弱，尤其不理解监督学习、无监督学习和神经网络。我希望两周内掌握人工智能导论的核心知识，每天可以学习 1 小时，喜欢通过案例、图解和代码来学习。
```

预期结果：生成不少于 6 个画像维度，包括知识基础、学习风格、薄弱点、学习目标、时间偏好、课程进度。

### 5.3 知识库管理

测试步骤：

1. 进入知识库管理页面；
2. 点击重建知识库；
3. 输入“监督学习和无监督学习有什么区别？”进行检索。

预期结果：

- 文档数为 10；
- 知识片段数大于 0；
- 检索返回课程来源；
- 向量模型不可用时显示关键词降级模式，系统仍可使用。

### 5.4 多智能体资源生成

测试步骤：进入学习资源页面，点击生成资源。

预期结果：生成以下资源中的至少 5 类，本项目应生成 6 类：

- 课程讲解文档；
- 分层练习题；
- 拓展阅读；
- 思维导图；
- 代码实操案例；
- 教学视频/动画任务。

### 5.5 学习路径规划

测试步骤：进入学习路径页面，点击生成学习路径。

预期结果：输出阶段目标、学习顺序、资源推荐、练习方式、评估指标和参考依据。

### 5.6 智能答疑

输入：

```text
请用通俗语言解释监督学习和无监督学习的区别，并给我一个自测题。
```

预期结果：

- 有生成进度或伪流式输出；
- 回答包含通俗解释；
- 包含图解说明和自测题；
- 展示课程来源标签；
- 内容审核通过。

### 5.7 学习效果评估

题目：

```text
监督学习和无监督学习有什么区别？
```

答案：

```text
监督学习使用带标签数据进行训练，目标是学习输入到输出的映射，常见任务包括分类和回归。无监督学习使用无标签数据，主要用于发现数据内部结构，例如聚类和降维。
```

预期结果：

- 返回评分和反馈；
- 更新知识点掌握度；
- 生成下一步学习任务；
- 动态更新学生画像中的薄弱点或课程进度。

### 5.8 多智能体架构展示

测试步骤：进入“多智能体架构”页面。

预期结果：展示 11 个智能体角色、协作流程和赛题要求对应关系。

## 6. 非功能测试

| 测试项 | 验收标准 |
| --- | --- |
| UI 美观性 | 页面统一、卡片化展示、层次清楚 |
| Markdown 渲染 | 答疑、资源、路径内容可正常渲染 |
| 长任务体验 | 资源生成、答疑具有进度条或步骤提示 |
| 防幻觉 | 答疑和路径内容展示知识来源 |
| 内容安全 | 输入输出经过审核或 MOCK 模式保持安全 |
| 提交完整性 | 源码、知识库、文档、部署说明齐全 |

## 7. 测试结论

经测试，系统已具备 A3 赛题要求的核心功能：对话式画像、多智能体资源生成、学习路径规划、智能答疑、学习效果评估、知识库管理、防幻觉引用与内容安全机制，可作为初赛作品提交和演示。
