# 部署说明

## 1. 项目结构

```text
A3_Learning_Agent/
├── backend/                 # Flask 后端
├── frontend/                # Vue 前端
├── rag_data/source_docs/    # 人工智能导论课程知识库
├── docs/                    # 提交文档
├── README.md                # 项目说明
└── 提交清单.md              # 初赛提交检查清单
```

## 2. 后端环境准备

推荐 Python 3.10+。

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

如使用 Conda：

```bash
conda create -n a3-learning-agent python=3.10 -y
conda activate a3-learning-agent
cd backend
pip install -r requirements.txt
```

## 3. 后端环境变量

复制环境模板：

```bash
copy .env.example .env
```

或手动创建 `backend/.env`：

```env
APP_HOST=0.0.0.0
APP_PORT=5000
APP_DEBUG=true

MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=a3_learning_agent
MYSQL_CHARSET=utf8mb4

XFYUN_APP_ID=你的讯飞APP_ID
XFYUN_API_KEY=你的讯飞API_KEY
XFYUN_API_SECRET=你的讯飞API_SECRET
XFYUN_SPARK_URL=wss://spark-api.xf-yun.com/v3.5/chat
XFYUN_SPARK_DOMAIN=generalv3.5

SEEDANCE_API_KEY=你的SeeDance密钥
SEEDANCE_API_URL=你的SeeDance接口地址
CONTENT_AUDIT_API_KEY=你的内容审核密钥
CONTENT_AUDIT_API_URL=你的内容审核接口地址

RAG_EMBEDDING_MODEL=BAAI/bge-small-zh-v1.5
MOCK_AI=true
JWT_SECRET_KEY=a3_learner_2026_softcup
```

说明：

- 正式接入讯飞服务时设置 `MOCK_AI=false`；
- 演示或离线环境可设置 `MOCK_AI=true`；
- RAG 向量模型不可用时系统会自动降级为关键词检索。

## 4. 数据库初始化

创建数据库并导入表结构：

```bash
mysql -u root -p < backend/db/init.sql
```

后端启动时也会自动补齐扩展表，包括学习事件、练习结果、掌握度和资源反馈表。

## 5. 启动后端

```bash
cd backend
python app.py
```

健康检查：

```bash
curl http://localhost:5000/health
```

## 6. 前端环境准备

推荐 Node.js 18+。

```bash
cd frontend
npm install
npm run dev
```

浏览器访问：

```text
http://localhost:5173
```

构建生产包：

```bash
npm run build
```

## 7. 知识库初始化

系统已内置《人工智能导论》10 章课程资料，路径：

```text
rag_data/source_docs
```

启动系统后进入“知识库管理”页面，点击“重建知识库”。

成功状态：

- 检索模式为 `向量`：表示 Chroma + BGE 向量检索可用；
- 检索模式为 `关键词`：表示向量模型不可用，已自动降级，演示仍可正常进行。

## 10. 常见问题

### 10.1 知识库重建失败

通常是模型下载失败或向量库文件被占用。当前版本已提供关键词降级模式，刷新页面后再次点击重建即可。

### 10.2 AI 接口无响应

检查 `.env` 中讯飞密钥是否正确。如果只是演示，可设置：

```env
MOCK_AI=true
```

### 10.3 数据库连接失败

检查 MySQL 是否启动、账号密码是否正确、数据库是否已创建。
